Capabilities

Everything in One Command

DevopsAutomate handles every step of the delivery workflow so your team can focus on building, not on admin.

🤖

AI Decomposition

GPT-4o translates a plain-language intake into a complete Epic → Feature → User Story → Task tree with acceptance criteria and story points.

🔷

Azure DevOps Integration

Every work item is created in your ADO project with correct parent-child links, metadata, priority, and area/iteration paths — via the Azure DevOps SDK.

📐

Standards Enforcement

Validates Gherkin acceptance criteria, story-point caps (max 13), Definition of Done checklist, task types, and branch naming conventions before any items are committed.

🧪

QA Automation

Generates happy-path, edge-case, and negative test cases per user story, then creates a full ADO Test Plan and Test Suites for the epic.

🐙

GitHub Integration

Creates standards-compliant feature branches (feature/{ado-id}-{slug}) and draft PRs with a DoD checklist linked back to ADO work items.

⚡

Single CLI Command

One devops-automate intake command drives the entire pipeline. Supports dry-run mode, flag overrides, and outputs a portable hierarchy.json.

GitHub-Driven Automation

Engineering Standards

Quality Enforced by Default

Every generated work item is validated against configurable rules in config/standards.yaml before being committed to Azure DevOps.

📋 Work Item Rules

  • Minimum 20-character descriptions
  • User stories must contain As a / I want / So that
  • Story point cap of 13 per user story
  • Allowed task types: Development, Testing, Documentation, DevOps, Design, Code Review

✅ Definition of Done

  • Code reviewed and approved
  • Unit tests written and passing
  • Acceptance criteria met
  • QA test cases created and passing
  • Documentation updated if applicable
  • No critical/high security vulnerabilities
  • PR linked to Azure DevOps work item

🐙 GitHub Standards

  • Branch naming: {type}/{ado_id}-{slug}
  • Types: feature, bugfix, hotfix, chore, docs
  • PR title: [#{ado_id}] {title}
  • Auto-label: ado-linked

🧪 QA Standards

  • Minimum 2 test cases per user story
  • Test types: Happy path, Edge case, Negative/error
  • Auto-creates ADO Test Plan & Test Suites
  • Acceptance criteria in Gherkin format
